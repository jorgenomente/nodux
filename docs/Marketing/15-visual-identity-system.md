# NODUX — Visual Identity System

Versión: v1.0  
Estado: Fundacional  
Propietario: NODUX  
Última actualización: 2026

---

# Propósito de Este Documento

Este documento define el sistema de identidad visual de NODUX.

Su objetivo es asegurar que todos los elementos visuales de la marca comuniquen de forma coherente el posicionamiento del producto.

La identidad visual debe transmitir que NODUX es:

una infraestructura operativa para retail.

No debe percibirse como:

- un POS básico
- un software administrativo genérico
- una plataforma tecnológica abstracta

Debe sentirse como **un sistema moderno que organiza la operación de las tiendas**.

---

# Principios Visuales de la Marca

La identidad visual de NODUX se basa en cuatro principios.

---

## 1 — Claridad Operativa

El diseño debe ser claro, limpio y estructurado.

Debe transmitir orden y control.

Esto refleja el propósito central del producto:

convertir la operación caótica de una tienda en un sistema visible.

Características visuales:

- interfaces claras
- layouts estructurados
- jerarquía visual fuerte
- espacios amplios
- tipografía legible

La sensación general debe ser:

control, claridad y organización.

---

## 2 — Estética de Infraestructura

NODUX no debe verse como una app ligera de consumo.

Debe sentirse como **infraestructura operativa del negocio**.

Esto implica una estética cercana a:

- dashboards profesionales
- sistemas de gestión
- plataformas operativas

La identidad visual debe transmitir:

estabilidad  
confiabilidad  
estructura

---

## 3 — Visualización de Sistemas

El concepto visual central de NODUX es:

**la operación del retail funcionando como un sistema conectado.**

Muchos elementos visuales pueden reforzar esta idea.

Ejemplos:

diagramas de flujo

Mercancía  
↓  
Inventario  
↓  
Ventas  
↓  
Caja  
↓  
Compras  
↓  
Pagos  
↓  
Reposición  

También pueden usarse:

- nodos conectados
- flujos operativos
- mapas de operación

Esto refuerza el concepto de **Retail Operating System**.

---

## 4 — Retail Real

Las imágenes y visuales deben mostrar **retail real**.

No imágenes genéricas de software.

Ejemplos recomendados:

- estanterías de tienda
- productos reales
- cajas registradoras
- equipos trabajando en tienda
- inventario en estantes

Esto conecta el producto con la realidad operativa del cliente.

---

# Paleta de Colores

La paleta debe transmitir:

claridad  
confianza  
operación

Propuesta base:

Color primario  
Azul profundo

Representa:

control  
estructura  
confiabilidad

Color secundario  
Verde operativo

Representa:

movimiento  
actividad  
operación

Colores neutros

- gris claro
- gris oscuro
- blanco

Estos colores permiten construir interfaces limpias.

---

# Tipografía

La tipografía debe ser:

moderna  
legible  
operativa

Recomendaciones:

Sans-serif limpia.

Ejemplos adecuados:

Inter  
Roboto  
Satoshi  
Helvetica Neue

Características importantes:

- alta legibilidad
- buena jerarquía
- claridad en dashboards

---

# Estilo de Interfaces

Las interfaces del producto deben convertirse en parte del lenguaje visual de la marca.

Características clave:

- dashboards claros
- indicadores visuales simples
- alertas visibles
- tablas ordenadas
- métricas fáciles de leer

Los screenshots del producto deben mostrar:

- inventario
- ventas
- alertas de vencimiento
- control multi-sucursal

Esto refuerza el valor operativo del sistema.

---

# Sistema de Iconografía

Los íconos deben representar procesos operativos del retail.

Ejemplos de iconos clave:

inventario  
ventas  
caja  
proveedores  
compras  
vencimientos  
sucursales  
pedidos online  

El estilo debe ser:

- simple
- geométrico
- consistente

Los iconos ayudan a visualizar el flujo operativo.

# Visualización del Comercio Omnicanal

Otro concepto visual clave es la conexión entre:

tienda física  
y  
tienda online.

Esto puede representarse visualmente con diagramas simples:

Cliente  
↓  
Tienda Online  
↓  
Sucursal más cercana  
↓  
Pedido listo para retirar

Este modelo representa:

Buy Online  
Pick Up In Store (BOPIS)

La identidad visual debe mostrar que:

la tienda física y el canal digital funcionan como una sola operación.

---

# Uso de Capturas del Producto

Las capturas del sistema deben mostrar escenarios reales.

Ejemplos:

inventario por sucursal

alertas de vencimiento

sugerencias de compra

control de caja

pedidos online por sucursal

Esto ayuda a que el visitante visualice rápidamente el valor del sistema.

---

# Estilo de Ilustraciones

Si se utilizan ilustraciones, deben ser:

- simples
- geométricas
- estructurales

Ejemplos:

diagramas de operación

mapas de flujo

sistemas conectados

No se recomiendan ilustraciones caricaturescas o demasiado informales.

---

# Estilo Fotográfico

Las fotografías deben reflejar:

retail real  
operación diaria  
personas trabajando en tienda

Evitar:

- bancos de imágenes demasiado artificiales
- escenas corporativas irreales
- imágenes tecnológicas abstractas

La marca debe sentirse cercana al negocio real.

---

# Principio de Diseño Fundamental

Cada elemento visual debe reforzar una idea:

**la tienda funciona como un sistema.**

Cuando el visitante ve la marca NODUX debe percibir:

orden  
estructura  
control  
operación conectada

---

# Declaración Final

La identidad visual de NODUX no representa una aplicación.

Representa **la infraestructura operativa de una tienda moderna**.

El objetivo del sistema visual es reforzar constantemente la idea central del producto:

NODUX es el sistema operativo de la tienda.
